# 더미-히든 시나리오는    더미(보여질미끼영상) 와 히든(숨길목적영상)을 하나의 스트림으로 만들어주는 시나리오 입니다

fakeke_enc_dec.py 파일을 실행

작동 모드는 입력 인자에 따라 3가지 모드로 동작 함
# 변조 시			: bin , bin -> bin 생성됨
# 복조 시			: bin -> yuv , bin 생성됨
# 변조여부 판단 시		: 변조 여부가 출력됨



입력 인자는

# 변조 시!! (입력인자 2 or 3개)
# 첫번째 인자: 더미 동영상 경로
# 두번째 인자: 히든 동영상 경로
# 세번째 인자(선택): 모드 ( 1 , 2 )
#                    1: 더미에 맞추기 ( 더미가 히든보다 길 경우 히든을 반복, 더미가 히든보다 짧을 경우 히든을 트림 )
#                    2: 히든에 맞추기 ( 히든이 더미보다 길 경우 더미를 반복, 히든이 더미보다 짧을 경우 더미를 트림 )
#            적지않을시: 양측 보존 ( 둘중 짧은영상을 반복하여 긴 영상 길이에 맞춤 )

# 복조 시!! (입력인자 2개)
# 첫번째인자 : 변조된 더미-히든 영상 경로
# 두번째인자 : 모드 (무조건 1 적을 것)

# 변조여부 판단 시!! (입력인자 1개)
# 첫번째인자 : 변조된 더미-히든 영상 경로


#             변조 시:  .bin파일 , .bin파일	->   .bin파일 생성
#             복조 시:  .bin파일		->   .yuv파일 , .bin파일 생성
# 변조여부 판단 시: .bin파일		->    변조여부가 커멘드라인 메세지로 출력

# 변조 시 ex)    fakeke_enc_dec.py C:\aaa\bbb\ccc.264  C:\aaa\bbb\ddd.264		->    ccc_ddd.264 생성
# 복조 시 ex)    fakeke_enc_dec.py C:\aaa\bbb\ccc.264  1                       		->    ccc_rev.264,  ccc_rev.yuv 생성
# 판단 시 ex)    fakeke_enc_dec.py C:\aaa\bbb\ccc.264                          		->    숨겨진 영상이 존재합니다.  or  숨겨진 영상이 존재하지 않습니다.  출력됨




#### 사용하는 외부 파일들
ldecod.exe :  h.264의 표준 jm 디코더, 264를 디코딩하여 yuv 파일 출력 시 사용함
ffmpeg.exe :  263 & mpeg2의 yuv 파일 출력 시 사용함
